CalendarBuddy

Python automation platform that saves my team ~80 hours a week of manual data collection at Socket Fiber.

Four core parts: a FastAPI + Redis + Celery + PostgreSQL web interface, plus three async Playwright modules imported into the same Docker environment — the Scraper (pulls installation jobs with a 16-job concurrent worker pool), the Spreader (a VRPTW solver that builds optimized technician routes from imported configuration), and the Assigner (routes returned jobs to the correct technician and contracting company). My team uses it multiple times a day, and it has almost completely automated my job. Taught me everything I know about CI/CD, deployment best practices, automated linting, and why everything ends up as a web app.

ActionJacksonInstalls.com

A Next.js + React frontend on top of an Express/EJS-routed backend — and an experiment in AI-driven development.

Built mostly by AI agents while I acted as Software Architect. ChatGPT produced the PRD and MVP; Claude Code CLI wrote the codebase; Codex ran an automated security and bug scan on every commit. A commit webhook forces the linter, type check, and Claude’s built-in security review to run before any commit can land. Biggest takeaway: AI coding agents are much further from “replacing” software engineers than the hype suggests — common failure modes included bland UIs, messy codebases, fabricated documentation, tests that didn’t actually test anything, and broken authentication.

BrowseShield

Mizzou IT Capstone — a browser extension that rates unsafe emails, websites, downloads, and links in real time.

Team capstone for my 4-person group, MAD J. A web-based browser extension plus a companion portal that uses custom heuristics and machine learning to score safety as you browse. Users can review their historical safety ratings, check existing passwords against known breaches, and generate secure replacements; parental controls are a stretch goal. My first real team software project — it made me significantly more disciplined about commit messages, branching, and pull-request etiquette — and my first browser extension.
